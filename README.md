# Linux Learning Journal

This repo documents my hands-on journey learning Linux, with a focus on
systems administration and cybersecurity fundamentals. It's meant to be a
transparent, dated record of what I studied, built, broke, and fixed —
not a polished tutorial.

## Why this exists

- Track real progress over time (not just certificates)
- Force myself to write clearly about technical problems and solutions
- Give future employers something concrete to review beyond a resume line
  that says "familiar with Linux"

## Structure

| Folder | What goes here |
|---|---|
| `/logs` | Dated journal entries — what I did, what broke, what I learned |
| `/scripts` | Bash/Python scripts written while practicing, with comments |
| `/writeups` | Longer-form writeups (CTF rooms, home-lab projects, incident simulations) |
| `/cheatsheets` | Command references and notes I built for myself |
| `/configs` | Dotfiles, hardening configs, server setups (sanitized — no secrets/IPs) |

## How I use this

Each log entry follows a simple format: what I set out to do, what actually
happened, what I learned, and what's next. See `/logs` for the template
and an example entry.
