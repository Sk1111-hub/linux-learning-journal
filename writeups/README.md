# Writeups

Longer-form pieces — CTF room walkthroughs (TryHackMe/HTB), home-lab
projects, or simulated incident investigations.

Unlike `/logs` (short daily entries), these are polished enough to share
as standalone posts and are good candidates to cross-post to a blog
(dev.to, Hashnode, GitHub Pages).

Suggested format:
1. **Objective** — what the room/project asked for
2. **Recon** — what you found and how
3. **Exploitation / Solution** — what you did, with commands and reasoning
4. **Root cause / Lesson** — the underlying concept, not just "I got the flag"
5. **Remediation** (if applicable) — how you'd fix/harden this in a real environment

Note: For TryHackMe/HTB, don't publish writeups for *active* machines —
check their rules on public writeups before posting.
